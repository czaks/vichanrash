Welcome to the Rash Quote Management System system guide and MySQL table setup. This is the first release of our program to the world, so please bear with the shoddy coding and whatever bugs you may find. It would be wonderful if you submitted any bugs to our Sourceforge website. For those of you who don't know what Rash is, it's a quote management system that is the best copy of Bash.org's layout I could achieve with minimal PHP skills.

1. Open config.php in a text editor, yes, we have to edit some variables and such to make it work, what a shame!
2. If you need to, change the value of $hostname from localhost to something else, be sure to keep whatever it is enclosed in single quotes. Hostname is the host of your mysql server, in most cases (if it's hosted on the same server as your web server) it will be localhost.
3. Change the value of $username to the name that you use to gain entry to your database. So if your login name for your database is Joe, you would put Joe inbetween the single quotes.
4. $dbpasswd is the password used in conjunction with the username that gets your entry to your database.
5. $dbname is the name of the database that you are logging into, or if there is more than one database under that username, the one you want rash to use.
6. $subtable should be left alone unless you want to test something, or for some reason want the submit quotes table to be something other than rash_submit
7. $quotetable is the table used to store all the quotes that are approved in the administration section, you may change the name of that table by editing this variable
8. $rashusers is the table that stores the users that you approve to use your administration section, change it's name here if you want to
9. $newstable is the table that stores your front page news, change the value of the variable to change the name of the table
10. $email is the e-mail address that shows up on the front page to email problems about YOUR site to
11. The customizations below $email should be edited after you get the system working and running. You can change things like colors, fonts, and errors in the customizations.

You have that all done? Good! We're going to proceed to the MySQL tables installation. After that you should delete this and the tables.php files from your server to ensure no one can do anything malicious with your setup.
